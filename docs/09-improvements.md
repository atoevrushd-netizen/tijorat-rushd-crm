# 09 — Анализ улучшений (что добавить для профессиональной CRM)

> Анализ по состоянию: MVP (Фазы 0–8) готов, баги аудита исправлены. Сейчас CRM **функциональна, но проста**.
> Здесь — чего не хватает, чтобы она стала **удобной, профессиональной и современной**.
> Это бэклог для обсуждения; реализуем по согласованному плану. Дата: 2026-06-22.

---

## A. Удобство и UX (быстрые победы, высокий эффект)

| # | Что | Зачем | Сложность |
|---|-----|-------|-----------|
| A1 | **Тосты-уведомления** (успех/ошибка) | Сейчас действия проходят молча — нет обратной связи | низкая |
| A2 | **Подтверждение удаления** (модалка) для задач/пользователей/вкладок/ссылок | Сейчас удаление мгновенное — легко стереть случайно | низкая |
| A3 | **Фильтр и поиск задач** (по статусу/типу) + пагинация/виртуализация | 60+ задач рендерятся разом — тяжело и неудобно | средняя |
| A4 | **Массовое создание задач** (напр. «30 Reels» одной кнопкой) | ТЗ прямо про 30+30 — вручную больно | средняя |
| A5 | **Сортировка** в списке пользователей и задач | Профессиональный must-have | низкая |
| A6 | Закрытие модалок по **Esc**, отправка по **Enter**, фокус-трап | Доступность и скорость | низкая |
| A7 | **Состояния пустых данных** с подсказками, скелетоны загрузки | Аккуратнее, понятнее новичку | низкая |
| A8 | **Мобильная навигация** (бургер-меню в TopBar) | «Устройства у всех разные» | низкая |

## B. Функциональность CRM (то, что делает её «настоящей»)

| # | Что | Зачем | Сложность |
|---|-----|-------|-----------|
| B1 | **Дашборд админа** (главная): сводка — пользователей, задач по статусам, лидов, последняя активность | Сейчас админ сразу попадает в список; нет «обзора» | средняя |
| B2 | **Раздел «Лиды»** (UI): просмотр/поиск лидов, конверсия лида → пользователь | Таблица `leads` есть, но UI нет; ТЗ акцентирует лиды | средняя |
| B3 | **Уведомления пользователю** (бейдж/лента): «вам отправлена задача», «задача принята» | Современная CRM не молчит; закрывает цикл работы | средняя |
| B4 | **Комментарии-тред к задаче** (вместо одного поля admin/user) | Переписка по задаче — частый сценарий | средняя |
| B5 | **Файлы к задаче** (загрузка в Storage, не только ссылки) | Удобнее, чем только URL | средняя |
| B6 | **Дедлайны: подсветка просроченных, «горит срок», сортировка по дедлайну** | Контроль сроков — суть работы | низкая |
| B7 | **Самостоятельное редактирование профиля** пользователем (имя, телефон, фото) | Разгружает админа (бэкенд уже это разрешает) | низкая |
| B8 | **Роли и права шире admin/user** (manager/operator) + **панель разработчика** | Из 1-го сообщения владельца: отдельная панель контроля | высокая |
| B9 | **Расширенный аудит**: логировать вход, изменения профиля, достижения (не только задачи) | Полная история «кто что» | низкая |
| B10 | **Глобальный поиск** (пользователи/задачи/лиды из одного места) | Скорость работы команды | средняя |
| B11 | **Экспорт** в CSV/Excel (пользователи, задачи, отчёты) | Отчётность для бизнеса | средняя |
| B12 | **Контент других вкладок** (продажи, KPI, финансы…) — типы полей вкладки через `tabs.schema` | Сейчас наполнена только «Медиа/Контент» | высокая |

## C. Техническое качество и надёжность

| # | Что | Зачем | Сложность |
|---|-----|-------|-----------|
| C1 | **Авто-генерация типов из Supabase** (`supabase gen types`) | Сейчас типы пишем руками — риск рассинхрона со схемой | низкая |
| C2 | **Оптимистичные обновления** (React Query) для действий с задачами | Мгновенный отклик UI | средняя |
| C3 | **Тесты компонентов и E2E** (Testing Library + Playwright) | Сейчас только юнит-тесты чистой логики | средняя |
| C4 | **CI** (GitHub Actions: lint + test + build на пуш) | Не пропускать поломки | низкая |
| C5 | **i18n** (react-i18next) | ТЗ упоминал таджикский; сейчас русский «вшит» | средняя |
| C6 | **Мониторинг ошибок** (Sentry) для прод-версии | Видеть сбои у пользователей | низкая |
| C7 | **Мягкое удаление/архив** вместо жёсткого `delete` | Защита от потери данных | средняя |
| C8 | **Пагинация ленты истории** (сейчас лимит 50) | Масштаб | низкая |
| C9 | **Доработать стабильность тестов** (Vitest изредка флакает на холодном старте) | Чистый зелёный прогон | низкая |
| C10 | **Хардненинг безопасности**: тип файла в Storage-политике, CORS Edge Function на домен | Защита прод-версии | низкая |

## Рекомендованные приоритеты (предложение к обсуждению)

**Этап «Удобство» (быстро и заметно):** A1 тосты · A2 подтверждения · A3 фильтр+пагинация задач · A4 массовое создание · B6 дедлайны · B7 самостоятельный профиль.

**Этап «Контроль» (для команды):** B1 дашборд админа · B2 лиды · B3 уведомления · B9 аудит шире · A8 мобильная навигация.

**Этап «Глубина» (крупное):** B4 комментарии-тред · B5 файлы · B8 роли/панель разработчика · B12 контент новых вкладок · B10 поиск · B11 экспорт.

**Этап «Качество» (фон, параллельно):** C1 типы · C2 оптимистичность · C3 тесты · C4 CI · C10 хардненинг.

> Дизайн (цвета/анимации) придёт с **UI Kit** владельца — он наложится на этот функционал.
