# 10 — Редизайн UI «Nebula UI» (тёмная тема) — план по фазам

> Источник: `UI KIT/UI Kit для CRM системы/` (`DESIGN-SYSTEM.md`, `tokens.css`, эталоны
> `UI Kit.dc.html`, `CRM Dashboard.dc.html`). Задача владельца: **полностью переделать дизайн** CRM
> в этот стиль — красиво, с анимациями, но **очень оптимизированно**. Логику не ломаем.

## Суть стиля (кратко)
- **Тёмная тема**, near-black фон `#0B0C0E`, прохладный нейтральный оттенок, разделение **тонкими
  границами**, а не блоками. Тени — только на приподнятом/модалках.
- **Один акцент — лайм `#C6F24E`**, точечно (главное действие, активный пункт, ключевая метрика).
- **Шрифты:** Hanken Grotesk (UI) + **JetBrains Mono для цифр/денег/ID/счётчиков**.
- **Сетка 4px, щедрые отступы, чёткая иерархия.** Радиусы: поля 12, карточки 16, панели/модалки 22, кнопки/бейджи — pill.
- **Микро-анимации сдержанно:** hover-lift, ступенчатое появление (`rise`), рост столбцов, donut-дорисовка,
  shimmer-скелетоны, ring-пульс. Тайминги 120/180/250ms, `cubic-bezier(.2,.8,.2,1)`. Уважать `prefers-reduced-motion`.
- **Запрещено:** светлая тема, белый фон, эмодзи в UI, border-left-accent карточки, тяжёлые тени везде.
- Эталон экрана: **сайдбар 248px + sticky-топбар (blur) + контент 28–32px**.

## Как внедряем технически (оптимизированно)
1. **Токены → Tailwind.** Переменные из `tokens.css` кладём в `src/styles/index.css`; в `tailwind.config.js`
   мапим их в тему (цвета `bg/surface/surface-2/3/elevated/border/text/accent/info/warn/danger` + `-soft`,
   радиусы, `fontFamily` sans=Hanken/mono=JetBrains, тени `sh-1/sh-2/glow`, анимации `rise/spin/pulse/ring/shimmer/barGrow`, ease).
   Дальше стилизуем компоненты Tailwind-утилитами поверх токенов — единый источник, Tailwind вычищает лишнее (лёгкий CSS).
2. **Шрифты** — `<link>` с `preconnect` + `display=swap` (позже можно self-host для idеальной скорости).
3. **Иконки** — `lucide-react` (outline, stroke ~1.75) вместо эмодзи; tree-shaking, лёгкие.
4. **Тёмная тема — единственная** (по требованию кита). Производительность: lazy-маршруты уже есть; анимации на CSS/transform; reduced-motion.

## Фазы редизайна (D1–D8)

> Каждая фаза: результат → визуальная проверка в браузере → обновление `07-progress.md` → проверка владельцем.
> Сначала тема и компоненты, потом экран за экраном (низкий риск, быстрый «вау»).

### D1 — Тема и фундамент
Интеграция `tokens.css` (переменные, фон-градиент, скроллбар, ::selection, keyframes, reduced-motion);
подключение шрифтов; расширение `tailwind.config.js` под токены; тёмная база приложения.
*Готово, когда:* приложение на тёмном фоне, ничего не сломано, build зелёный.

### D2 — Библиотека компонентов (Nebula primitives)
Button (primary/secondary/outline/ghost/danger + размеры + loading) · Input (label/focus-ring/иконка/ошибка) ·
Select · Checkbox/Radio/Toggle · Badge+точка · Chip · Avatar (squircle, лайм-градиент инициалы) · Card (+hover-lift) ·
Modal (elevated+blur) · Spinner · Skeleton (shimmer) · Banner/Alert (4 типа) · Pagination · Tabs · SegmentedControl ·
StatusBadge (статусы задач/пользователей в kit-стиле). Подключить `lucide-react`.
*Готово, когда:* все базовые компоненты соответствуют `UI Kit.dc.html`.

### D3 — Каркас: Sidebar + Topbar (AppShell)
Сайдбар 248px (лого squircle+glow, поиск, группы пунктов mono-лейблами, активный=accent-soft, счётчики-pill,
профиль-карточка + выход) + sticky-топбар (blur, заголовок+дата, уведомления, primary-кнопка).
Адаптив: на мобильном сайдбар → drawer (бургер).
*Готово, когда:* админ-разделы в фирменном каркасе; кабинет — облегчённый вариант.

### D4 — Экран входа
Центр-карточка на тёмном с лайм-свечением, поля/кнопки по киту, задел под переключатель вход/регистрация.

### D5 — Админ: список пользователей + карточка
Таблица в kit-стиле (mono-заголовки UPPERCASE, ячейка клиента avatar+имя+email, статус-бейджи, ⋯, hover-строка),
поиск + фильтр-чипы, пагинация; карточка пользователя (шапка, табы, секции); модалки (создание/редактирование/пароль).

### D6 — Задачи «Медиа/Контент» + вкладки
Задачи как kit-карточки (статус-бейджи, ссылки, даты, прогресс); табы в kit-стиле; формы.

### D7 — Личный кабинет = дашборд
KPI-карточки (TaskStats → KPI: mono-числа, дельта), donut прогресса задач, профиль-шапка, задачи, достижения —
по эталону `CRM Dashboard.dc.html`.

### D8 — Анимации, полировка, оптимизация
`rise` (ступенчатый вход) · hover-lift везде · скелетоны-shimmer для загрузок · ring для уведомлений ·
donut/bar анимации · `prefers-reduced-motion`. Оптимизация: загрузка шрифтов, проверка бандла/CSS, Lighthouse-смоук,
**чек-лист соответствия из `DESIGN-SYSTEM.md` §8**.
