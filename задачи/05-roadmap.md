# 05 — Дорожная карта (Roadmap)

Строим **по фазам**. Одна фаза = один заход (сессия). В конце каждой фазы:
работающий результат → проверка владельцем → коммит → обновить `docs/07-progress.md` → следующая.

> Claude Code: делай ТОЛЬКО текущую фазу. Не забегай вперёд.

---

## Фаза 0 — Фундамент
**Цель:** проект запускается локально и показывает страницу входа.

Сделать:
- Инициализировать Vite + React + TypeScript.
- Подключить Tailwind CSS.
- Настроить React Router, базовый layout, страницу 404.
- Создать `src/lib/supabase.ts` и `.env.example` (URL и anon key).
- Положить `logo-light.svg` в `public/logo/`, подключить как favicon и в шапку.
- Создать структуру папок из `docs/03-architecture.md`.

**Готово, когда:** `npm run dev` запускается, видна пустая страница входа с логотипом.

---

## Фаза 1 — Аутентификация и роли
**Цель:** можно войти, система знает роль и пускает в нужный раздел.

Сделать:
- Миграция: таблица `profiles` (+ enums `user_role`, `user_status`) и `leads`.
- Включить RLS и базовые политики (админ — всё, пользователь — своё).
- Вход через email/пароль и через Google OAuth.
- При регистрации записывать `leads` (имя + телефон обязательно).
- Защита маршрутов по роли: `admin` → админ-панель, `user` → личный кабинет.
- Logout.

**Готово, когда:** админ и пользователь логинятся и попадают каждый в свой пустой раздел.

---

## Фаза 2 — Управление пользователями (Админ)
**Цель:** админ создаёт и ведёт пользователей.

Сделать:
- Список пользователей: имя, фото, статус, кол-во задач (готово/в работе), дата регистрации.
- Поиск по пользователям.
- Создать пользователя (имя, телефон, направление, статус, логин, начальный пароль через Auth).
- Редактировать пользователя.
- Загрузка фото в Supabase Storage.
- Пагинация списка.

**Готово, когда:** админ создаёт, ищет, редактирует пользователей; список быстрый.

---

## Фаза 3 — Карточка пользователя
**Цель:** «медкарта» пользователя со всеми данными.

Сделать:
- Страница карточки: фото, имя, телефон, логин, статус, направление, дата регистрации,
  комментарий админа.
- Блок достижений (таблица `achievements`): просмотр + добавление админом.
- Заготовка блока истории активности.
- Каркас вкладок внутри карточки (пока пустые табы).

**Готово, когда:** открыв пользователя, админ видит полную карточку и табы.

---

## Фаза 4 — Гибкая система вкладок
**Цель:** вкладки управляются данными, легко добавлять новые.

Сделать:
- Миграция: таблица `tabs`, сидинг вкладки `media` («Медиа / Контент»).
- Универсальный рендерер: карточка читает активные вкладки из БД и строит табы.
- Админ-управление вкладками (вкл/выкл, порядок) — можно простое.

**Готово, когда:** табы в карточке приходят из таблицы `tabs`; добавление записи = новая вкладка.

---

## Фаза 5 — Вкладка «Медиа / Контент» + Задачи
**Цель:** полный рабочий цикл задачи.

Сделать:
- Миграции: `tasks`, `task_links` (+ enums статусов).
- Админ: создать задачу (название, тип reels/креатив, дедлайн, ссылки, комментарий),
  менять статус, отметить «отправлено пользователю».
- Пользователь: видит задачу, ссылки, даты; **принимает** или **отправляет на правку**
  с комментарием.
- Статусы и бейджи статусов; даты `created/sent/accepted`.
- Пример наполнения: 30 Reels + 30 креативов.

**Готово, когда:** админ ведёт задачи, пользователь принимает/отклоняет — полный цикл работает.

---

## Фаза 6 — История действий
**Цель:** видно, кто что и когда сделал.

Сделать:
- Запись в `activity_log` при ключевых действиях (создание задачи, смена статуса, принятие).
- Лента истории в карточке пользователя.

**Готово, когда:** действия фиксируются с датой и автором, видны в карточке.

---

## Фаза 7 — Личный кабинет пользователя (доводка)
**Цель:** пользователю всё понятно и удобно.

Сделать:
- Дашборд пользователя: его задачи, достижения, что принято, что в процессе, комментарии команды.
- Простой чистый интерфейс, мобильная адаптивность.

**Готово, когда:** пользователь видит цельную картину своего пути.

---

## Фаза 8 — Оптимизация и качество
**Цель:** быстро на слабых устройствах, стабильно.

Сделать:
- Ленивая загрузка страниц, оптимизация списков и запросов.
- Состояния загрузки/ошибок/пустых данных.
- Проверка RLS-политик на безопасность.
- Чистка кода, проверка размеров файлов.

**Готово, когда:** приложение лёгкое, без явных тормозов и багов.
