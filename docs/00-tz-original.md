## 1. Цель программы
Нужно создать внутреннюю программу/CRM, где мы сможем вести всех наших пользователей.
Пользователь — это наш студент/бизнесмен.
У каждого пользователя должна быть своя личная карточка, как медицинская карта в больнице,
где хранится вся информация о нём, его прогресс, выполненные задачи, достижения, материалы и история работы.

## 2. Роли в системе
**Администратор** — должен иметь доступ ко всем пользователям. Возможности:
- создавать нового пользователя;
- редактировать информацию пользователя;
- загружать фото пользователя;
- выдавать логин и пароль;
- добавлять задачи, вкладки и материалы;
- отмечать выполненную работу;
- видеть статус принятия работы пользователем;
- добавлять ссылки, даты и комментарии.

**Пользователь** — должен иметь личный кабинет. Возможности:
- входить по логину и паролю;
- видеть свою карточку;
- видеть свои достижения;
- отмечать свои успехи;
- принимать или не принимать выполненные задачи;
- видеть даты, ссылки и материалы;
- оставлять комментарий к задаче.

## 3. Карточка пользователя
У каждого пользователя — отдельная карточка. Начальная информация:
фото; имя и фамилия; номер телефона; логин; пароль; статус пользователя;
направление бизнеса; дата регистрации; комментарий администратора;
достижения пользователя; история активности.
В будущем структура карточки должна расширяться.

## 4. Вкладки внутри карточки пользователя
В карточке должно быть несколько вкладок. На первом этапе нужна вкладка по медиа/маркетингу.

**Вкладка: Медиа / Контент** — ведём задачи по созданию контента. Например: 30 Reels; 30 креативов.
Для каждой задачи:
- название задачи; тип задачи: Reels или креатив; дата создания; дедлайн; статус выполнения;
  ссылка на готовый материал; комментарий администратора; статус принятия пользователем;
  дата принятия пользователем; комментарий пользователя.

Статусы задачи: не начато; в работе; готово; отправлено пользователю; принято пользователем; требуется правка.

Администратор отмечает, что задача выполнена. Пользователь у себя в кабинете подтверждает, что принял эту работу.

## 5. Даты и история
Все действия должны сохраняться с датой. Нужно фиксировать:
когда задача была создана; когда была выполнена; когда отправлена пользователю;
когда пользователь её принял; кто внёс изменение.

## 6. Ссылки на материалы
У каждой задачи должно быть место для ссылки. Например: Google Drive; видео; дизайн; пост; другой материал.

## 7. Гибкая структура на будущее
Программа должна быть сделана так, чтобы в будущем можно было добавлять новые вкладки. Например:
обучение; продажи; финансы; диагностика бизнеса; домашние задания; отчёты; встречи; документы;
результаты; KPI; стратегия.
Важно, чтобы программист сразу заложил гибкую архитектуру, чтобы новые вкладки можно было добавлять
без полной переделки системы.

## 8. Личный кабинет пользователя
Пользователь после входа должен видеть: свою информацию; свои задачи; свои достижения;
выполненные материалы; что он принял; что ещё в процессе; комментарии от команды.
Интерфейс должен быть простым и понятным.

## 9. Админ-панель
В админ-панели — список всех пользователей. Для каждого видно:
имя; фото; статус; количество выполненных задач; количество задач в работе; дату регистрации;
быстрый переход в карточку пользователя. Также нужен поиск по пользователям.

## 10. Главные требования
Программа должна быть: простой в использовании; удобной для команды; безопасной;
с логином и паролем для каждого пользователя; с возможностью расширения; с историей действий;
с понятными статусами задач; с доступом администратора ко всем карточкам.

## 11. Первый этап разработки
На первом этапе нужно сделать:
- Регистрацию пользователей.
- Логин и пароль для пользователя.
- Карточку пользователя.
- Место для фото.
- Основную информацию о пользователе.
- Админ-панель.
- Вкладку «Медиа / Контент».
- Задачи на 30 Reels и 30 креативов.
- Статусы задач.
- Место для ссылок.
- Подтверждение задачи пользователем.
- Даты всех действий.

## 12. Общая идея
Мы хотим получить программу, где каждый студент/бизнесмен будет видеть свой путь, свои задачи,
свои достижения и результаты. А наша команда сможет удобно вести каждого пользователя,
контролировать работу, отмечать выполненные задачи и видеть весь процесс в одном месте.
