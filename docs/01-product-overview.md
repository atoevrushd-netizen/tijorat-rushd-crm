# 01 — Обзор продукта (Product Overview)

## Название
**Tijorat & Rushd CRM** — внутренняя CRM-система компании.

## Одна фраза
Система, где каждый студент/бизнесмен видит свой путь, задачи, достижения и результаты,
а команда удобно ведёт каждого пользователя в одном месте.

## Главная идея
У каждого пользователя есть личная **карточка**, похожая на медицинскую карту в больнице:
в ней хранится вся информация о нём, его прогресс, выполненные задачи, достижения,
материалы и история работы.

- **Команда (админы)** — ведут пользователей, ставят задачи, отмечают выполненное,
  видят весь процесс.
- **Пользователь** — заходит в личный кабинет, видит свои задачи и достижения,
  принимает или отклоняет выполненную работу, оставляет комментарии.

## Кто пользователи
- **Администратор** — сотрудник команды Tijorat & Rushd. Полный доступ ко всем пользователям.
- **Пользователь** — студент/бизнесмен. Видит только свой кабинет и свою карточку.

## Что важно владельцу
1. **Гибкость на будущее.** Сейчас одна вкладка (Медиа/Контент), но архитектура должна
   позволять добавлять новые вкладки (обучение, продажи, финансы, диагностика, KPI и т.д.)
   без переделки системы.
2. **Производительность.** Много пользователей, разные устройства — система должна быть лёгкой.
3. **Простота интерфейса.** Понятно и удобно для команды и для пользователей.
4. **История действий.** Все важные действия фиксируются с датой и автором.
5. **Безопасность.** У каждого свой логин/пароль; роли разграничивают доступ.

## Этапы (крупно)
- **Этап 1 (MVP):** регистрация, роли, карточка пользователя, админ-панель,
  вкладка «Медиа/Контент» с задачами (30 Reels + 30 креативов), статусы,
  подтверждение задач пользователем, история и даты.
- **Этап 2+:** новые вкладки, продвинутый UI Kit, аналитика, оптимизация.

## Чего НЕ делаем на старте
- Не делаем сложный дизайн/анимации (придёт позже с UI Kit).
- Не делаем мобильное приложение (только веб).
- Не реализуем все вкладки сразу — только Медиа/Контент как первую.
