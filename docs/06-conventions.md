# 06 — Соглашения по коду (Conventions)

## Язык
- Код, имена переменных, типы, поля БД — **на английском**.
- Тексты интерфейса — на таджикском/русском (как укажет владелец позже).
- Комментарии можно на русском.

## Именование
- Компоненты: `PascalCase` → `UserCard.tsx`
- Хуки: `useCamelCase` → `useUserTasks.ts`
- Функции/переменные: `camelCase`
- Типы/интерфейсы: `PascalCase` → `type Task`, `type UserProfile`
- Поля БД и enums: `snake_case`
- Файлы api: `userApi.ts`, `taskApi.ts`

## Размер и структура файлов
- Цель: ≤ ~150–200 строк на файл. Больше — разбивай.
- Один компонент = один файл. Логику выноси в хуки.
- Никаких «мега-файлов»: код должен легко перечитываться целиком.

## TypeScript
- Строгий режим включён. Без `any` (если без него никак — комментарий с причиной).
- Типы данных держим в `types.ts` фичи или в `src/types`.
- Типы — это документация модели данных.

## Данные и Supabase
- Все запросы к Supabase — через функции в `*/api/*.ts`.
- Использование в компонентах — через хуки на TanStack Query.
- Не дублируем строки подключения; один клиент в `src/lib/supabase.ts`.
- Никаких секретов в коде — только через `.env` (и `.env.example` без значений).

## Состояния UI
- Каждый экран обрабатывает: загрузка / ошибка / пусто / данные.
- Списки — с пагинацией или бесконечной прокруткой.

## Git / процесс
- Маленькие осмысленные коммиты с понятным сообщением.
- В конце фазы — коммит + обновление `docs/07-progress.md`.

## Стиль (на старте)
- Простой, чистый, минималистичный (Tailwind).
- Не вкладываемся в анимации/сложный дизайн до получения UI Kit.
- Адаптивность под мобильные — да, базовая.

## Чего избегать
- Не рефакторить несвязанные части ради «красоты».
- Не вводить новые библиотеки без необходимости и без отметки в `progress.md`.
- Не менять стек без подтверждения владельца.
