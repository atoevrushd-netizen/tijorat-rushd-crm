## Вход по логину + пароли для админа (2026-06-23)
- **Вход по логину (username)** вместо email: при создании — поле «Логин»; email генерируется в фоне
  как `<логин>@users.tijorat.local` (Supabase Auth требует email). Старые аккаунты (admin, студенты)
  входят по своему email. Карточка показывает «Логин». Helper — `src/lib/loginEmail.ts`.
- **Админ видит выданные пароли открыто:** таблица `user_credentials` (миграция `0012`, RLS — только админ);
  заполняется Edge Function при создании и смене пароля. В карточке — поле «Пароль» (показать/копировать).
  ⚠️ Открытые пароли в БД — сознательный компромисс для внутренней CRM; при переезде в облако помнить о риске.

## Фаза Б — Настройки и сохранность данных (2026-06-23)
Решения владельца: бэкап = скачивание файла; импорт = безопасное слияние (ничего не удаляет); защита от потери = «Корзина» (мягкое удаление).
- **Шаг Б1 ✅:** панель **«Настройки»** (`/settings`, для админа и лида, содержимое по роли; пункт в обоих меню).
  - Лид: простые настройки — профиль + информация об аккаунте (логин, статус, подписка).
  - Админ: профиль + **«Создать бэкап»** — скачивает JSON со всеми данными CRM (лиды, пользователи,
    задачи, вкладки, достижения, история, пароли). Это главная защита «не потерять лидов» (только чтение).
- **Шаг Б2 ⬜ (следующий):** импорт из бэкапа (безопасное слияние) + «Корзина» удалённых лидов
  (мягкое удаление + восстановление). Destructive — с отдельным ревью.
