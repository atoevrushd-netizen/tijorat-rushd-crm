# Документация — Tijorat & Rushd CRM

Это каноническая документация проекта. Главный файл правил — корневой `../CLAUDE.md`
(его Claude Code читает автоматически каждую сессию). Здесь — детали.

## Порядок чтения

1. [`00-tz-original.md`](00-tz-original.md) — исходное ТЗ заказчика (дословно).
2. [`01-product-overview.md`](01-product-overview.md) — что за продукт и зачем.
3. [`02-requirements.md`](02-requirements.md) — полные функциональные требования.
4. [`03-architecture.md`](03-architecture.md) — стек, структура папок, принципы.
5. [`04-data-model.md`](04-data-model.md) — таблицы БД, связи, RLS-безопасность.
6. [`05-roadmap.md`](05-roadmap.md) — план разработки по фазам (Фаза 0 → 8).
7. [`06-conventions.md`](06-conventions.md) — правила кода и именования.
8. [`07-progress.md`](07-progress.md) — живой журнал: что сделано, что дальше.
9. [`08-open-questions.md`](08-open-questions.md) — открытые вопросы к владельцу.
10. [`09-improvements.md`](09-improvements.md) — бэклог улучшений для профессиональной CRM.
11. [`10-ui-redesign.md`](10-ui-redesign.md) — редизайн «Nebula UI» (тёмная тема), план по фазам.

## Принципы документации

- **Каждый шаг — обновляем `.md`.** В конце фазы правим `07-progress.md`; при новых решениях — соответствующий документ.
- **`/docs` — канонично.** Папка `../задачи/` заморожена как исходник спецификации заказчика.
- **Документы короткие и по делу**, чтобы их можно было быстро прочитать целиком.
