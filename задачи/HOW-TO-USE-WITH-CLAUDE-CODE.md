# Как использовать это с Claude Code

Привет, Равшан. Тут — порядок действий и готовые промпты. Копируй по очереди.

---

## Шаг 1. Положи файлы в проект

Скопируй всё это в корень папки проекта:
```
твой-проект/
├── CLAUDE.md
├── docs/
│   ├── 01-product-overview.md
│   ├── 02-requirements.md
│   ├── 03-architecture.md
│   ├── 04-data-model.md
│   ├── 05-roadmap.md
│   ├── 06-conventions.md
│   └── 07-progress.md
└── public/logo/logo-light.svg   ← положи сюда свой логотип
```

`CLAUDE.md` Claude Code читает сам в начале каждой сессии. Остальное — по ссылкам из него.

---

## Шаг 2. Первый промпт — «прочитай и подтверди» (БЕЗ кода)

> Прочитай `CLAUDE.md` и все файлы в папке `docs/`. Пока **не пиши код**.
> Когда прочитаешь — кратко перескажи своими словами: что мы строим, какой стек,
> как мы работаем и какие фазы. Если что-то непонятно или противоречит — задай вопросы.
> Подтверди, что готов начать с Фазы 0.

Это проверяет, что он всё понял правильно, до того как что-то делать.

---

## Шаг 3. Запуск фаз — по одной

Когда он подтвердил, запускай фазы по очереди. На каждую — отдельный промпт:

> Начинаем **Фазу 0** из `docs/05-roadmap.md`. Сделай **только её**.
> Сначала покажи короткий план (3–6 пунктов), потом выполняй.
> В конце: обнови `docs/07-progress.md`, сделай коммит и **остановись** — я проверю.

После проверки — следующая:

> Фаза 0 принята. Начинаем **Фазу 1**. Так же: только эта фаза, сначала план,
> потом код, в конце — обнови `progress.md` и остановись.

И так до Фазы 8.

---

## Шаг 4. Если что-то идёт не так

- **Пишет слишком много сразу / лезет вперёд:**
  > Стоп. Делай только текущую фазу. Откати лишнее, вернись к плану из roadmap.

- **Файл стал огромным:**
  > Этот файл слишком большой. Разбей по правилам из `docs/06-conventions.md`.

- **Запутался в коде:**
  > Перечитай связанные файлы перед изменением. Покажи план, потом меняй.

- **Забыл контекст (новая сессия):**
  > Прочитай `CLAUDE.md` и `docs/07-progress.md`, чтобы понять, на чём мы остановились.

---

## Почему так, а не один большой промпт

1. **CLAUDE.md** — Claude Code читает его автоматически каждый раз. Это его «память» о проекте.
2. **Фазы** — он не «устаёт», но в больших задачах легко теряет фокус. Маленькие фазы =
   меньше ошибок, проще проверять, легче откатывать.
3. **progress.md** — между сессиями он забывает контекст. Этот файл возвращает его в курс дела.
4. **read-first** — заставляем читать перед изменением → меньше поломок в большом коде
   (твоя главная боль).

## Что подготовить тебе заранее

- **Логотип** `logo-light.svg` → в `public/logo/`.
- **Supabase-проект:** создай проект на supabase.com, возьми Project URL и anon key —
  они понадобятся в `.env` на Фазе 1. (Claude Code подскажет точные шаги.)
- **Google OAuth** (для входа через Google) — настраивается в Supabase, можно на Фазе 1.

---

## Когда дойдёшь до красоты

После Фазы 8 (рабочий MVP) дашь свой **UI Kit**. Тогда промпт:

> MVP готов и работает. Вот UI Kit (цвета, шрифты, анимации): [...].
> Применяй его постепенно, по одному экрану, не ломая логику. Начни с экрана входа.
